# /re-init — Initialize Rating Extractor for a Carrier

## Purpose

Run this inside a carrier folder (e.g., `E:\intelli-new\Cssi.Net\Portage Mutual`).
It reads the master catalog built by `/re-index`, matches the carrier name from the
current directory, finds the relevant manuals, and sets up `.re-workstreams/` so all
other `/re-*` commands work.

**Prerequisite:** `/re-index` must have been run first to build the master catalog.

## Trigger

Slash command: `/re-init`

No arguments needed — the carrier is detected from the current working directory.

Optional flags:
- `--refresh` — re-run init, overwriting existing `.re-workstreams/`

## Inputs

- Current working directory (must be a carrier root folder)
- `{plugin_root}/knowledge/manual-catalog.yaml` — master catalog from `/re-index`

## Outputs

All outputs go to `{carrier_root}/.re-workstreams/`:

1. **`paths.md`** — All absolute paths (mandatory first-read for other commands)
2. **`config.yaml`** — Carrier metadata, matched manuals, carrier structure

## Steps

### Step 1: Pre-flight Checks

1. **Determine plugin root** — resolve from THIS skill file's location:
   ```
   plugin_root = go up from skills/re-init/SKILL.md → rating-extractor/
   ```

2. **Determine carrier root** — use the current working directory:
   ```
   carrier_root = CWD
   carrier_name = last folder name in CWD (e.g., "Portage Mutual")
   ```

3. **Check for master catalog:**
   ```
   Read {plugin_root}/knowledge/manual-catalog.yaml
   ```
   If not found:
   ```
   "No manual catalog found. Run /re-index {path_to_pdf_folder} first to scan your manuals."
   ```
   Stop.

4. **Check for existing `.re-workstreams/`:**
   - If exists and `--refresh` not passed: "Already initialized. Use `/re-init --refresh` to re-initialize, or run `/re-query` directly."
   - If exists and `--refresh` passed: proceed (will overwrite)

### Step 2: Match Carrier to Manuals

1. **Read the master catalog** and search for this carrier:
   - Exact match on carrier name key (e.g., "Portage Mutual")
   - Fuzzy match on aliases (e.g., CWD says "Portage Mutual", catalog has alias "Portage")
   - Case-insensitive, ignore trailing spaces

2. **If match found**, extract the list of manuals for this carrier:
   ```
   Found carrier: Portage Mutual
   Matched manuals:
     1. portage-auto-rate-2025-04       — OMAP Auto Rate Manual (269 pages, Auto, ON)
     2. portage-residential-2025-12     — Residential Pro Manual (118 pages, Residential, ON)

   Proceed with these manuals?
   ```
   Wait for confirmation.

3. **If NO match found:**
   ```
   Carrier "Portage Mutual" not found in the master catalog.

   Available carriers:
     - Portage Mutual (3 manuals)
     - Unica (2 manuals)
     - The Commonwell (2 manuals)
     - Red River (1 manual)

   Options:
     1. Did you mean one of the above? Tell me which.
     2. Run /re-index to re-scan your manuals folder.
   ```
   Wait for developer input.

### Step 3: Discover Carrier Structure

Scan the carrier folder to understand its layout (used by `/re-bridge`):

1. **List province folders** (Ontario, Alberta, Manitoba, etc.)
2. **For each province**, list LOB folders (Auto, Home, Condo, etc.)
3. **Find latest dated subfolder** per LOB (highest YYYYMMDD)
4. **Locate Code/ directories**
5. **Extract carrier prefix** from .vbproj filenames (e.g., "PORT")

If the folder doesn't look like a TBW carrier, warn but continue:
```
Note: This doesn't look like a standard TBW carrier folder (no province/LOB structure found).
/re-query will work fine, but /re-bridge code mapping won't be available.
```

### Step 4: Discover External Tools

1. **vb-parser.exe** — needed for `/re-bridge`:
   - Check `.iq-workstreams/paths.md` in carrier root → look for `vb_parser` path
   - Check `.iq-update/tools/win-x64/vb-parser.exe` relative to carrier root
   - Check PATH: `where vb-parser.exe`
   - If found: record path
   - If not found: record "NOT_FOUND" (warning will be shown by `/re-bridge`)

2. **Python:**
   - Check `python --version`
   - Record the working command

### Step 5: Write Configuration Files

1. **Create `{carrier_root}/.re-workstreams/` directory**

2. **Write `paths.md`:**
   ```markdown
   # Rating Extractor — Resolved Paths
   # Generated by /re-init on {date}
   # Carrier: {carrier_name}
   # Every /re-* command reads this file FIRST.

   | Key | Path |
   |-----|------|
   | plugin_root | {absolute path to rating-extractor/} |
   | carrier_root | {absolute path to carrier folder} |
   | carrier_name | {carrier name} |
   | workstreams | {carrier_root}/.re-workstreams |
   | manual_catalog | {plugin_root}/knowledge/manual-catalog.yaml |
   | manual_reader_agent | {plugin_root}/agents/manual-reader.md |
   | code_mapper_agent | {plugin_root}/agents/code-mapper.md |
   | contract_registry | {plugin_root}/contracts/contract_registry.yaml |
   | build_index_tool | {plugin_root}/tools/build_index.py |
   | python_cmd | {python or python3} |
   | vb_parser | {path or NOT_FOUND} |
   | manual:{slug} | {absolute path to each matched PDF} |
   | toc:{slug} | {absolute path to each toc.yaml in plugin_root/knowledge/manual-index/} |
   ```

3. **Write `config.yaml`:**
   ```yaml
   carrier_name: "Portage Mutual"
   carrier_root: "E:\\intelli-new\\Cssi.Net\\Portage Mutual"
   initialized_at: "2026-03-12T14:00:00Z"

   manuals:
     - slug: "portage-auto-rate-2025-04"
       title: "OMAP Auto Rate Manual"
       lob: "Auto"
       province: "ON"
       effective_date: "2025-04-01"
       source_path: "C:\\manuals\\2025 04 01 Auto Rate Manual (posted 2025 01 24).pdf"
       total_pages: 269
     - slug: "portage-residential-2025-12"
       title: "December 2025 ON Residential Pro Manual"
       lob: "Residential"
       province: "ON"
       effective_date: "2025-12-01"
       source_path: "C:\\manuals\\December 2025 ON Residential Pro Manual.pdf"
       total_pages: 118

   # Carrier structure (for /re-bridge)
   provinces:
     - code: "ON"
       name: "Ontario"
       lobs: ["Auto", "Home", "Condo", "Tenant"]
       code_dir: "E:\\intelli-new\\Cssi.Net\\Portage Mutual\\Ontario\\Code"

   carrier_prefix: "PORT"

   tools:
     vb_parser: "E:\\...\\vb-parser.exe"  # or "NOT_FOUND"
     python_cmd: "python"
   ```

### Step 6: Present Summary

```

Manuals:
  - portage-auto-rate-2025-04       (269 pages, Auto, ON)
  - portage-residential-2025-12     (118 pages, Residential, ON)

Carrier structure:
  - Ontario: Auto, Home, Condo, Tenant
  - Code root: E:\intelli-new\Cssi.Net\Portage Mutual\Ontario\Code

Tools:
  - vb-parser: Found
  - Python: 3.11.4

Ready:
  /re-query "What is the genNow! discount?"
  /re-bridge "What is the genNow! discount?"
```

## Error Handling

| Error | Action |
|-------|--------|
| Master catalog not found | "Run `/re-index {path}` first." |
| Carrier not in catalog | Show available carriers, ask developer |
| No manuals match carrier | Show catalog contents, suggest re-running `/re-index` |
| `.re-workstreams/` exists | Suggest `--refresh` |
| Carrier structure invalid | Warn, continue (only affects `/re-bridge`) |
| vb-parser not found | Record NOT_FOUND, `/re-bridge` will degrade gracefully |
